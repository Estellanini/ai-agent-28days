# Source Priority

## 1. Official

Examples:

- Official documentation.
- Official product blog.
- GitHub release notes.
- Specification websites.

Use for factual claims about API behavior, model availability, versions, pricing, and product capabilities.

## 2. Papers / Technical Reports

Use for research claims, benchmark results, architecture ideas, and limitations.

Check:

- publication date;
- benchmark setup;
- whether code/data are available;
- whether claims are evaluated or only proposed.

## 3. Engineering Blogs

Use for implementation lessons and production case studies.

Watch for:

- missing details;
- marketing bias;
- narrow use cases.

## 4. Media Reports

Use for business context and ecosystem changes.

Do not use as the only source for technical details.

## 5. Community Discussion

Use for pain points, anecdotes, and tool comparisons.

Never treat as verified fact without a primary source.
