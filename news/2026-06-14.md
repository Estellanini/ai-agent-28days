## 资讯 3：OX Security 披露 MCP 架构性 RCE，150M+ 下载受影响（研究或安全风险）

- 来源：[OX Security 研究博客（含 10+ CVE 安全公告）](https://www.ox.security/blog/the-mother-of-all-ai-supply-chains-critical-systemic-vulnerability-at-the-core-of-the-mcp/)；另见 [arXiv 2506.13538 大规模 MCP server 实证研究](https://arxiv.org/html/2506.13538v2)
- 发布日期：约 2026 年 5–6 月（CVE-2026-30xxx 系列；精确发布日期未在搜索结果给出，标注待核实）
- 一句话事实：OX Security 发现 MCP（Anthropic 维护的 agent 通信标准）官方 SDK 里有一个架构性缺陷，能通过 UI 注入 / 零点击 prompt injection / 恶意市场包等 4 类路径触发任意命令执行（RCE）；影响 150M+ 下载、约 20 万实例、6 个生产平台；已产出 10+ CVE（LiteLLM、LangChain、Windsurf 等均在列），但 Anthropic 称该行为"by design"、拒绝在协议层修复。
- 和 AI Agent 的关系：safety / MCP（正是 Day01 学的概念）
- 我今天学到的判断：MCP 让工具接入"丝滑"，但代价是把信任边界扩大到所有第三方 MCP server。一旦 agent 能调带副作用的工具，prompt injection 就从"说错话"升级成"被远程执行代码"——这是 tool / MCP 拥有副作用后最危险的一类失败点。
- 对本项目的行动：agent-system-map 里画的"Policy / Approval"和"Runtime / Sandbox"两道闸，正是防这类 RCE 的关键；自己接 MCP server 时只用官方 registry、放进沙箱、限制可执行命令。Day01 的 `agent-security-model` / `workspace-boundary` 要重点读。
- 可信度：工程博客 / 安全研究（OX Security，含 CVE 编号，接近一手研究）；"Anthropic 拒绝修复"等表述需以 Anthropic 官方回应为准。附带：该文提到"上周 Anthropic 发布 Claude Mythos 用以保护全球软件安全"，与资讯 1 的 Mythos 发布时间线吻合。
